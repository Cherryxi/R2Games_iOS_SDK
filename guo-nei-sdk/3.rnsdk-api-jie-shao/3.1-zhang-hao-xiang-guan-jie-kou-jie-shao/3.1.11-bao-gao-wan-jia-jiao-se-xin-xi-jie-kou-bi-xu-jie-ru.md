# 3.1.11 报告玩家角色信息接口\(必须接入\)

#### 功能说明：

 研发方可以调用该接口上报玩家角色信息。请在登录完成以及创建角色的时候，调用该接口。

#### 接口形式：

```objectivec
-(void)reportRoleWithServerId:(NSString*)serverId roleId:(NSString*)roleId roleName:(NSString*)roleName roleLevel:(NSString*)roleLevel roleCreateTime:(NSString*)roleCreateTime lastLoginTime:(NSString*)lastLoginTime extraData:(NSString*)extraData Handler:(RNGameRoleHandler)handler;
```

#### 接口参数：

 serverId: 游戏服id，必须。

 roleId : 角色id，必须。

 roleName: 角色名，非必须，首次创角时必传；重复上报时，可不传。

 roleLevel: 角色等级，非必须，可不传。

 roleCreateTime: 创角时间戳，非必须，首次创角时，不传默认当前时间。

 lastLoginTime: 上次登录时间，非必须，首次创角时，不传默认当前时间。

 extraData:透传字段，如需使用则传，不需使用可传空字符串。

 handler：回调，取result.data 中的json字符串，根据需求自行解析相应字段。

#### 接口示例：

```objectivec
[[RNSDKApi sharedInstance]reportRoleWithServerId:@"1" roleId:@"2"  roleName:@"ww" roleLevel:@"10" roleCreateTime:@""  lastLoginTime:@"" extraData:@""  Handler:^(int code, NSString *msg, RNGameRoleResponse *result) {
           if (code == RN_RESPONSE_SUCCESS_CODE) {
            //上报角色信息成功，具体的状态码在result.data中，若不详请联系SDK服务器端技术人员
                   NSLog(@"上报角色信息成功，data %@ ",result.data);
               }else{
                   //上报角色信息失败
                   NSLog(@"失败，code %d ,msg%@",code,msg);
               }
       }];
```
